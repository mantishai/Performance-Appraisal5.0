## 核心需求

### 需求1：新增员工自动分配工号并创建登录账号

**业务逻辑**：
- 在【组织架构】或【员工管理】中新增员工时，**自动生成工号**作为系统登录名
- **默认密码**统一设为 `1`（或 `123456`，可根据你现有逻辑决定）
- 新增员工的同时，自动在 `users` 表中创建对应的登录账号

**工号生成规则**（建议）：
- 格式：`EMP` + 日期(YYYYMMDD) + 3位流水号
- 例如：`EMP20260510001`
- 或简化为：年份后两位 + 4位序号，如 `260001`

**数据库操作**：
- 在新增员工的 API 中（如 `POST /api/employees`），使用**数据库事务**同时操作 `employees` 表和 `users` 表
- `employees` 表存储员工详细信息（姓名、部门、岗位等）
- `users` 表存储登录信息（username=工号，password=加密后的默认密码）

**后端示例逻辑**：
```javascript
// POST /api/employees
async function createEmployee(req, res) {
    const connection = await db.getConnection();
    await connection.beginTransaction();
    
    try {
        // 1. 生成工号
        const employeeNo = await generateEmployeeNo();
        
        // 2. 插入 employees 表
        const [result] = await connection.execute(
            'INSERT INTO employees (employee_no, name, department, position, ...) VALUES (?, ?, ?, ?, ...)',
            [employeeNo, req.body.name, req.body.department, req.body.position, ...]
        );
        
        // 3. 自动创建登录账号（users表）
        const hashedPassword = await bcrypt.hash('1', 10); // 默认密码加密
        await connection.execute(
            'INSERT INTO users (username, password, employee_id, role, display_name) VALUES (?, ?, ?, ?, ?)',
            [employeeNo, hashedPassword, result.insertId, 'employee', req.body.name]
        );
        
        await connection.commit();
        res.json({ success: true, employeeNo: employeeNo });
    } catch (error) {
        await connection.rollback();
        res.status(500).json({ success: false, message: error.message });
    } finally {
        connection.release();
    }
}

javascript
async function changePassword(req, res) {
    const userId = req.session.userId; // 从session获取当前用户ID
    const { oldPassword, newPassword } = req.body;
    
    // 1. 验证旧密码
    const [rows] = await db.execute('SELECT password FROM users WHERE id = ?', [userId]);
    const isValid = await bcrypt.compare(oldPassword, rows[0].password);
    if (!isValid) {
        return res.status(400).json({ success: false, message: '旧密码不正确' });
    }
    
    // 2. 更新密码
    const hashedPassword = await bcrypt.hash(newPassword, 10);
    await db.execute('UPDATE users SET password = ? WHERE id = ?', [hashedPassword, userId]);
    
    res.json({ success: true, message: '密码修改成功' });
}

数据库调整汇总
users 表结构（需要在 sql/ 目录的初始化脚本中添加）
sql
CREATE TABLE IF NOT EXISTS users (
    id INT PRIMARY KEY AUTO_INCREMENT,
    username VARCHAR(50) NOT NULL UNIQUE COMMENT '登录名（员工=工号，非员工=自定义）',
    password VARCHAR(255) NOT NULL COMMENT '加密后的密码',
    display_name VARCHAR(50) NOT NULL COMMENT '显示名称',
    role VARCHAR(20) DEFAULT 'employee' COMMENT '角色：admin/employee',
    employee_no VARCHAR(20) DEFAULT NULL COMMENT '关联工号（员工用户有值）',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_employee_no (employee_no)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
employees 表结构（确认包含工号字段）
sql
ALTER TABLE employees ADD COLUMN employee_no VARCHAR(20) NOT NULL UNIQUE COMMENT '工号（自动生成）';

注意事项

工号唯一性：工号生成逻辑需考虑并发情况，建议使用数据库自增或分布式ID

密码加密：使用 bcrypt 加密存储密码，默认密码 1 也需加密

Session管理：修改密码后，建议让用户重新登录或更新session

权限控制：管理员重置密码的接口必须验证管理员权限

前端联动：修改密码弹窗需集成到现有的 Toast 提示系统中

错误处理：所有API需返回统一的JSON格式 { success: boolean, message: string, data?: any }
