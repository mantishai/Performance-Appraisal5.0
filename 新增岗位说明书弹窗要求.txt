请帮我在当前开发的人力资源管理系统中实现"岗位说明书弹窗"功能。我会提供详细的参考逻辑、数据结构、样式要求和API规范。

---

## 一、功能概述

实现一个模态弹窗，用于查看和编辑各岗位的岗位说明书。弹窗支持两种模式：
1. **查看模式（只读）**：所有表单字段禁用，仅展示数据
2. **编辑模式**：所有表单字段可编辑，支持增删职责行和考核指标行，支持保存/恢复操作

---

## 二、弹窗触发入口

请为以下位置添加触发逻辑：

1. **个人看板页面** - "岗位说明书"按钮：
   - 点击触发：openReadOnlyPositionModal(positionId)
   - 模式：只读查看
   - positionId通过当前登录用户的岗位名称从getPositionIdByName()函数获取

2. **领导看板页面** - 侧边栏"岗位说明书"树状菜单：
   - 点击人员触发：openPositionModal(positionId)
   - 模式：可编辑
   - 树状菜单结构参考：部门列表下有岗位列表，每个岗位是一个可点击项
   - 岗位ID范围：hr1 到 hr75

3. **组织架构页面** - 人员详情中的"岗位说明书"按钮（如有）

5.7 数据持久化
javascript
// 保存到服务器
async function saveHrPositionDesc() {
    const positionData = {
        positionId: window.currentHrPositionId,
        data: 收集弹窗中所有表单数据()
    };
    
    try {
        const res = await fetch('/api/hr/position/save', {
            method: 'POST',
            headers: {
                'Content-Type': 'application/json',
                'x-session-id': sessionId
            },
            body: JSON.stringify(positionData)
        });
        
        if (res.ok) {
            保存到localStorage作为备份();
            showToast('✅ 保存成功');
            切换回只读模式();
        }
    } catch (e) {
        保存到localStorage();
        showToast('⚠️ 网络异常，已保存到本地');
    }
}

// 从服务器加载
async function loadHrPositionDesc() {
    try {
        const res = await fetch('/api/hr/position/' + window.currentHrPositionId, {
            headers: { 'x-session-id': sessionId }
        });
        
        if (res.ok) {
            const data = await res.json();
            应用数据到弹窗表单();
            return;
        }
    } catch (e) {}
    
    // 降级：从localStorage加载
    const localData = localStorage.getItem('hrPositions');
    if (localData) {
        应用数据到弹窗表单();
    }
}
六、API接口规范
6.1 保存岗位说明书
端点：POST /api/hr/position/save

请求头：Content-Type: application/json, x-session-id: {sessionId}

请求体：

json
{
    "positionId": "hr1",
    "data": {
        "info": { ... },
        "purpose": "...",
        "duties": [ ... ],
        "qualification": { ... },
        "conditions": { ... },
        "metrics": [ ... ],
        "documents": "..."
    }
}
响应：{ "success": true }

6.2 加载岗位说明书
端点：GET /api/hr/position/{positionId}

请求头：x-session-id: {sessionId}

响应：{ "success": true, "data": { "info": {...}, ... } }

6.3 降级策略
API调用失败时，自动降级到localStorage存储

localStorage key：'hrPositions'，值为JSON对象 { "hr1": {...}, "hr2": {...} }

加载时优先从API获取，失败后从localStorage读取

七、技术约束
不引入任何前端框架（React/Vue等），使用原生JavaScript

CSS不依赖外部库，使用项目已有的CSS变量和样式体系

代码风格：函数命名使用驼峰命名法，变量名语义化

复用项目现有组件：Toast提示（showToast函数）、认证工具（getHeaders函数）

兼容性：支持Chrome、Edge、Firefox最新版本

错误处理：所有异步操作需包含try-catch，网络失败有降级方案

八、特别注意
弹窗的positionId存储在 window.currentHrPositionId 全局变量中

标题动态更新：根据职位名称实时更新弹窗标题

恢复按钮：从API/localStorage重新加载数据覆盖当前表单

关闭弹窗时：如果有未保存修改，可以提示用户（可选）

导出PDF时：生成纯展示版HTML（无表单控件），使用表格展示数据

量化指标库部分的内容是固定的（5个模块的详细指标），不需要可编辑
